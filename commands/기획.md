# `/bini-zzang:기획` — 풀 기획 파이프라인

당신은 `pm-orchestrator` 에이전트를 활성화한다. `agents/pm-orchestrator.md` 파일의 정체성·원칙·동작을 로드해 그대로 수행한다.

## 사용자 입력

`$ARGUMENTS`에 사용자가 기획할 주제가 포함되어 있다. 없으면 물어본다:

```
어떤 걸 기획할까요?
예: "수요반응 DR 운영 플랫폼", "이커머스 주문 관리 시스템", "헬스케어 앱 기능 추가"
```

## 실행 단계 (pm-orchestrator 지시)

### 0. Kickoff
1. 주제 확인
2. Pre-gate 질문:
   - 도메인 (DR/SaaS/FinTech 등)
   - 타겟 사용자 세그먼트
   - 스코프 (모든 Phase? 일부만?)
   - 기존 자료 경로 (있으면)
   - 예상 완료 시간 (참고용)
3. `.plan/` 디렉토리 생성 (없으면)
4. `index.json` 초기화

### 1. Phase 0 — 용어집
- `glossary-writer` 호출
- 핵심 용어 5개 사용자 확인 (Mid-gate)
- `.plan/00_glossary.md` 생성

### 2. Phase 1 — 기반 분석
- `market-researcher` 호출 (순차 1)
- `.plan/_constants.json` + `.plan/01_market.md` 생성
- Mid-gate: 페르소나 시드·페인 검토
- `strategist` 호출 (순차 2)
- 포지셔닝 3안 제시 → 사용자 선택
- `.plan/02_strategy.md` 생성
- Post-gate: Non-Goals 확정 (이후 변경 금지)

### 3. Phase 2 — 사용자·구조
- `user-researcher` 호출 (먼저, 순차)
- `.plan/03_personas.md` 생성
- Mid-gate: 페르소나 검토
- `flow-designer`와 `ia-architect` **병렬 호출**
- `.plan/03_flows/FL-*.md`, `.plan/03_ia.md`, `.plan/03_role-matrix.csv` 생성

### 4. Phase 3 — 기능·데이터
- Pre-gate: 대상 L2 기능 범위 확인
- `feature-writer` 호출 (순차)
- `.plan/04_features/F-*.md` 생성
- `data-modeler` 호출 (순차)
- `.plan/04_data-model.md` 생성

### 5. Phase 4 — 지원 산출물
- `policy-writer`, `kpi-designer`, `risk-analyst` **병렬 호출**
- `.plan/05_policies.md`, `.plan/05_kpis.md`, `.plan/05_risks.md` 생성

### 6. Phase 5 — 정합성 검증
- `consistency-checker` 호출
- `.plan/06_consistency-report.md` 생성
- **Critical 위반 시 사용자 체크 대화**:

```
⚠ 정합성 검증에서 🔴 Critical 모순 N건 발견
[V-001] ...
  [1] ... (권장)
  [2] ...
  [3] 무시

→ 번호 선택:
```

사용자 응답 없이 절대 자동 수정하지 않음.

### 7. 완료 선언
- `index.json` 최종 갱신
- 산출물 요약 테이블 출력
- 다음 단계 추천

## 사용 예시

```bash
/bini-zzang:기획 수요반응 DR 운영 플랫폼
/bini-zzang:기획 SaaS 팀 워크플로우 앱
/bini-zzang:기획        # 주제 묻기
```

## 사용자 중단 대응

파이프라인 중간에 사용자가 "잠깐", "멈춰", "여기까지" 등을 말하면:
1. 현재 Phase의 진행 상황을 `index.json`에 저장
2. 마지막 성공한 Phase까지 완료된 것으로 선언
3. 나중에 `/bini-zzang:기획 resume`으로 재개 가능 (로드맵)

## 에스컬레이션

- 선행 아티팩트 파일 유효성 실패 → 중단 + 사용자 알림
- 동일 게이트 3회 "다시" → 방향성 재확인 요청
- Phase 5에서 critical 5건 이상 → 설계 재검토 권고

## 산출물 위치

사용자의 현재 작업 디렉토리 아래 `.plan/` 폴더.
