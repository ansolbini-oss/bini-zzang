# `/bini-zzang:기획-검증` — 정합성 재검증

`consistency-checker` 에이전트를 호출해 기존 `.plan/` 폴더의 모든 아티팩트를 교차검증한다.

## 사전 조건

- `.plan/` 폴더 존재
- 최소 `00_glossary.md` + Phase 1 이상의 아티팩트

## 실행 단계

1. `.plan/` 인덱싱
   - 모든 아티팩트 로드
   - 각 파일의 frontmatter 유효성 검사

2. `consistency-checker` 호출
   - 14개 규칙(C1~C14) 전수 실행
   - 심각도별 분류

3. 리포트 작성
   - `.plan/06_consistency-report.md` 생성/갱신

4. Critical 위반 시 사용자 체크
```
⚠ 정합성 검증 완료

🔴 Critical: N건 — **사용자 확인 필수**
🟠 Warning: N건
🟡 Info: N건

[V-001] C3 페르소나 참조 정합성
  위치: FL-003.md:8
  내용: P-005 참조하지만 personas.md엔 없음

  [1] user-researcher에게 P-005 정의 추가 요청 (권장)
  [2] FL-003의 primary_persona를 P-001~P-004 중 선택
  [3] 무시하고 계속 (사유 입력)

→ 번호 선택:
```

5. 선택 기록
   - `.plan/.decisions/06_consistency_<ISO>.json`

## 사용 시나리오

- Phase 간 실행 후 정기 체크
- 아티팩트 수동 수정 후 재검증
- 특정 에이전트 재실행 후 영향 확인
- 릴리스 전 최종 QA

## 완료 후

```
✅ 정합성 검증 완료
  • Critical: 0건 (또는 해결됨)
  • 리포트: .plan/06_consistency-report.md
  • 다음 단계 추천: (상황별)
```
